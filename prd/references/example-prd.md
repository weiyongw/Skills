# 批量批改任务 — 产品需求文档

> **版本**：v1.0 | **日期**：2025-09-20 | **作者**：产品团队

---

## 一句话说清楚

让测试用户在一个页面上传作业图片、填标准答案、发起批量批改，替代现在的逐张手动批改。

---

## 1. 背景与问题

### 现状是什么

目前 POC 测试用户要批改作业，需要一张张上传图片，每次只能改一张。测试 20 张图片要重复操作 20 遍，效率很低，测试人员反馈"太繁琐了"。

### 为什么现在要做

POC 阶段即将进入客户演示环节，批量能力是客户验收的硬性要求。如果不能批量批改，演示效果会大打折扣。

---

## 2. 目标

### 做成什么样

- 目标 1：用户一次最多提交 20 张图片，一键发起批改
- 目标 2：提交后自动跳转任务监控页，用户不需要手动刷新就能看到进度

### 怎么衡量成功

| 指标 | 当前值 | 目标值 | 说明 |
|------|--------|--------|------|
| 单次批改操作耗时 | 约 5 分钟/张 | 20 张 < 3 分钟（提交操作） | 从进入页面到提交成功的时间 |
| 提交成功率 | N/A（新功能） | > 95% | 排除网络异常等不可控因素 |

### 不做什么

- 不做：历史任务的配置复用（"用上次的设置再来一次"）
- 不做：批改结果的导出和分享
- 不做：多人协作批改

---

## 3. 用户与场景

### 目标用户

| 用户角色 | 特征描述 | 核心诉求 |
|----------|----------|----------|
| POC 测试用户 | 教育领域从业者，需要验证系统的批改准确率 | 快速提交大量图片，观察批改效果 |

### 核心使用场景

**场景 1：批量提交批改任务**
> 测试用户拿到一叠学生作业（约 10-20 张照片），需要一次性上传到系统，填入标准答案，选择批改模型后发起批改。目前要一张张操作，太慢了。

**场景 2：提交出错后修正**
> 用户上传了图片、填了答案，但提交时发现有张图片太大被拒了。需要能删掉那张重新传，而不是全部推倒重来。

---

## 4. 整体流程

用户进入批量任务页 → 上传图片 + 填标准答案 + 选配置 → 提交 → 跳转到监控页看进度。

```mermaid
flowchart TD
  A[进入批量任务页面] --> B[上传图片（最多 20 张）]
  B --> C[填写标准答案]
  C --> D[选择批改模型和配置]
  D --> E[点击"开始批改"]
  E --> F{校验通过？}
  F -->|是| G[创建任务，跳转监控页]
  F -->|否| H[显示具体错误提示]
  H --> I[用户修正后重新提交]
  I --> E
```

---

## 5. 需求详述

### 5.1 图片上传

**做什么**：用户通过拖拽或点击选择，上传待批改的作业图片。

**业务规则**：
1. 单次最多上传 20 张图片
2. 单张图片不超过 10 MB，总量不超过 150 MB
3. 支持的格式：JPG、PNG、BMP
4. 上传后显示文件列表，每项可单独删除

**异常与边界**：
| 异常情况 | 系统行为 | 用户看到什么 |
|----------|----------|------------|
| 图片超过 10 MB | 拒绝该张，其他正常 | 提示"xxx.jpg 超过 10 MB，请压缩后重试" |
| 超过 20 张 | 拒绝多出的 | 提示"单次批改最多 20 张" |
| 格式不支持 | 拒绝该文件 | 提示"仅支持 JPG/PNG/BMP 格式" |

**验收标准**：
- [ ] 拖拽上传 5 张 JPG 图片 → 文件列表显示 5 张，可逐个删除
- [ ] 上传第 21 张图片 → 被拒绝，提示"最多 20 张"，前 20 张不受影响
- [ ] 上传一张 15 MB 的图片 → 被拒绝，提示"超过 10 MB"

---

### 5.2 标准答案录入

**做什么**：用户在多行输入框中，按固定格式录入标准答案。

**业务规则**：
1. 格式为"序号. 单词"，每行一个（如 `1. apple`）
2. 系统实时校验：序号必须连续、内容必须是英文字符
3. 校验不通过时，在输入框下方显示具体错误，同时禁用"开始批改"按钮

**异常与边界**：
| 异常情况 | 系统行为 | 用户看到什么 |
|----------|----------|------------|
| 序号不连续（如跳过第 3 行） | 校验失败 | 提示"第 3 行序号缺失" |
| 包含中文字符 | 校验失败 | 提示"标准答案仅支持英文字符" |
| 输入为空 | 校验失败 | 提示"请输入标准答案" |

**验收标准**：
- [ ] 输入 `1. apple` + `2. banana` → 校验通过，按钮可用
- [ ] 输入 `1. apple` + `3. cherry`（跳过 2）→ 提示"第 2 行序号缺失"
- [ ] 清空输入框 → 提示"请输入标准答案"，按钮禁用

---

### 5.3 批改配置与提交

**做什么**：用户选择批改模型和参数，确认后提交任务。

**业务规则**：
1. 批改模型：下拉选择，有默认值
2. 智能终审：开关，默认开启
3. 权重模板：下拉选择，从系统加载
4. 全部校验通过后"开始批改"按钮才激活

**用户操作流程**：
1. 用户上传图片、填写答案、选择配置
2. 点击"开始批改"
3. 系统创建任务
4. 页面跳转到任务监控页，显示"处理中 0/N"

**异常与边界**：
| 异常情况 | 系统行为 | 用户看到什么 |
|----------|----------|------------|
| 创建任务失败（服务端错误） | 不跳转，保留输入 | 弹窗提示"任务创建失败，请重试" |
| 网络超时 | 不跳转，保留输入 | 提示"网络异常，请检查网络后重试" |

**页面布局**：
```text
+-------------------------------------------------------------+
| 批量任务创建                                                  |
+==============================+==============================+
| 图片上传（拖拽或点击选择）     | 标准答案                      |
|                              |                              |
| - hw_001.jpg        [删除]   | 1. apple                     |
| - hw_002.jpg        [删除]   | 2. banana                    |
| - hw_003.jpg        [删除]   | 3. cherry                    |
| ...                          | ...                          |
|                              |                              |
+------------------------------+------------------------------+
| 批改模型: [默认模型 v]  智能终审: [开]  权重模板: [默认 v]     |
+-------------------------------------------------------------+
| [取消]                                       [开始批改]       |
+-------------------------------------------------------------+
```

**验收标准**：
- [ ] 上传 10 张图片 + 合法答案 + 默认配置 → 点击"开始批改" → 跳转监控页，显示"处理中 0/10"
- [ ] 图片或答案有问题 → "开始批改"按钮灰色不可点
- [ ] 提交失败 → 弹窗提示错误原因，页面内容不丢失
