# 阶段04：内容产出

**目标**：根据框架写出一篇**经得起时间检验**的长青文章

---

## 核心理念

> **长青写作的秘诀：讲原理时像教授，讲案例时像朋友。**

原理要讲透，让读者真正理解；
案例要生动，让读者感同身受。

---

## 长青写作的五个要点

### 1. 讲原理，不讲信息
- ✓ "值得学的技能有一个共同特点..."
- ✗ "2024年最值得学的五个技能是..."

### 2. 用具体案例证明抽象原理
- ✓ 先说原理，再用案例证明
- ✗ 只讲案例，让读者自己总结

### 3. 给读者一个可带走的框架
- ✓ "用这三个问题判断：..."
- ✗ 讲完就讲完了，没有可操作的东西

### 4. 承认边界和局限
- ✓ "这个原理适用于...，但在...情况下不适用"
- ✗ 把观点说成绝对真理

### 5. 保留更新的空间
- ✓ "以 X 为例..."（案例可以换）
- ✗ "X 是最好的..."（断言会过时）

---

## 步骤

### 1. 读取框架文件

- 确认各部分的内容和占比
- 明确每段要讲什么
- 检查长青检查清单

---

### 2. 逐部分写作

#### 开篇写作

**目标**：抓住读者，提出问题

**长青开篇的技巧**：
- 用场景切入而非新闻切入（场景可以复用，新闻会过时）
- 提出一个"永恒的问题"而非"当下的热点"
- 快速点出核心矛盾

**示例对比**：
- ✗ "最近 ChatGPT 很火，大家都在讨论..."（过时）
- ✓ "每次新技术出现，都会有人问：这个要学吗？"（永恒）

**字数**：100-150字

---

#### 原理部分写作

**目标**：讲透核心道理，这是文章的灵魂

**长青原理写作的技巧**：

1. **先给结论**：核心原理是什么，一句话说清
2. **解释为什么**：这个原理为什么成立，底层逻辑是什么
3. **给出边界**：这个原理适用的范围，什么情况下不适用
4. **连接人性**：这个原理和什么人性基本面相关

**示例结构**：
```
[原理结论] — 一句话说清核心观点
[底层逻辑] — 为什么这个原理成立
[边界条件] — 什么情况下适用/不适用
[人性连接] — 和什么人性基本面相关
```

**字数**：300-400字

---

#### 案例部分写作

**目标**：用具体案例证明原理

**长青案例写作的技巧**：

1. **案例服务于原理**：先回顾原理，再展开案例
2. **正反对比**：有正例有反例，更有说服力
3. **跨领域案例**：如果可能，用不同领域的案例，增加可迁移性
4. **案例可更新**：用"以...为例"的表述，暗示案例可以换

**示例结构**：
```
[回顾原理] — 一句话回顾
[正面案例] — 具体、生动、有细节
[反面案例] — 对比，为什么不符合原理
[回扣原理] — 案例证明了什么
```

**字数**：300-350字

---

#### 框架/方法部分写作

**目标**：给读者一个可带走的思考工具

**长青框架写作的技巧**：

1. **简单易记**：三个要素左右，太复杂记不住
2. **可迁移**：能用在其他问题上
3. **可操作**：读者知道怎么用

**示例结构**：
```
[框架名称] — 起一个好记的名字
[框架要素] — 2-4个要素
[如何使用] — 一个简单的步骤
[迁移示例] — 这个框架还能用在哪
```

**字数**：150-200字

---

#### 结尾写作

**目标**：总结、呼应、给出下一步

**长青结尾的技巧**：

1. **重申核心观点**：一句话总结灵魂句
2. **呼应开篇**：和开篇的问题/场景形成闭环
3. **指向行动**：读者下一步可以做什么
4. **不要说教**：是邀请，不是命令

**字数**：50-100字

---

### 3. 写作风格指南

**语言风格**：
- 保持口语化，像在对话
- 用短句，易读
- 可以用"你"直接对读者说话
- 避免学术腔和官方腔

**长青语言技巧**：
- 避免时间词："现在"、"最近"、"今年" → 用"每当"、"总是"
- 避免具体工具名（除非作为案例）
- 避免绝对断言："最好的"、"唯一的" → 用"一个有效的"、"通常"

**示例**：
- ✗ "2024年，AI 发展很快..."
- ✓ "每一次技术浪潮来临时..."

- ✗ "ChatGPT 是最好的写作工具"
- ✓ "以 ChatGPT 为例，我们可以看到..."

---

### 4. 长青检查

**写完后，过一遍这个清单**：

| 检查项 | 问题 | 通过? |
|--------|------|-------|
| 原理 | 核心原理讲透了吗？ | □ |
| 案例 | 案例是否服务于原理？ | □ |
| 框架 | 读者能带走什么框架？ | □ |
| 时间 | 有没有会过时的表述？ | □ |
| 边界 | 有没有承认局限？ | □ |
| 独特 | AI 能写出一样的吗？ | □ |

---

### 5. 交付并迭代

**创建文章文件**：`{选题名}_article.md`

**告诉用户**：
- 文章已经写完，请看一下
- 特别关注：原理是否讲透、案例是否合适、有没有过时风险

**根据反馈修改**：
- 原理不够清楚 → 补充底层逻辑
- 案例不够生动 → 增加细节
- 有过时风险 → 调整表述
- 缺少框架 → 提炼可带走的工具

---

## 关键原则

- **原理优先**：原理讲透是第一位的
- **案例支撑**：案例是为原理服务的
- **时间意识**：随时检查有没有会过时的表述
- **读者视角**：读者能带走什么？

---

## 本阶段结束标志

- 文章已经写完
- 通过长青检查
- 用户确认满意
- 或根据反馈修改到满意为止
